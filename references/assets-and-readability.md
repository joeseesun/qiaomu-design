# 视觉资产与可读性规范（Assets & Readability）

> 来源：v3.0 用户三连击反馈（Hero 右侧简陋 / 作品集可读性差 / 多样性展示差）+
> taste-skill 4.4/4.5/4.8/4.9 深挖。凡页面含视觉资产或大段文字，本文件必读。

## 一、Hero 资产规范（"文字 + 渐变斑点不是 Hero，是占位符"）

分屏 Hero 的视觉侧、任何产品展示位，按优先级三选一：

1. **有图像生成工具就必须用**（generate-image / imagegen / fal 系列 / AI gateway
   图像模型）：按板块生成正确宽高比的真实资产——产品截图、场景摄影、纹理。
   不许因为"手写 CSS 更快"跳过此步
2. **高保真真实组件预览**（无图像工具或禁外图时）：把产品 UI 做成**真的迷你组件**，
   必须同时满足：
   - **深度 ≥ 3 层**：窗口壳/环境光影 + 主 UI + 浮动元素（悬浮标注、通知气泡、
     光标轨迹、副窗口），彼此有重叠与投影
   - **内容密度 ≥ 8 个真实元素**：真实微文案、有机数据、头像、状态点、时间戳——
     空心矩形 + 三条灰线是被禁的"div 假截图"
   - **至少 1 处活的细节**：呼吸状态点 / 打字光标 / 数字滚动 / 波形动画
   - **构图有姿态**：微旋转（1-3deg）、越界溢出、透视或堆叠，拒绝端正贴在栏里
3. **抽象编辑构图**：巨字排版 / 几何构成 / 分层渐变场景（多层 + 噪点 + 光源方向），
   作为一个完整的"场"来设计，而不是一个色块

**配套禁令**：logo 墙用真 SVG 标（simpleicons CDN）或为虚构品牌造字母标，禁纯文字
wordmark 行；logo 下不加行业小标签；引用语 ≤3 行且署名带姓名+职位。

## 二、可读性红线（美是加分项，读不了是死刑）

任何气质都不许牺牲这些：

- **正文对比度**：亮底 ≥ 4.5:1；暗底 ≥ 7:1 且正文用 `rgba(255,255,255,.85)`、
  次要文字**不得低于** `rgba(255,255,255,.60)`（纯黑底上的 #666 灰字是可读性事故）
- **氛围层让路**：噪点/网格纹理在**正文区**透明度 ≤ 3%（装饰区可到 6%）；
  纹理线不得穿过正文段落造成条纹干扰——必要时给正文容器一块干净底
- **按钮对比度审计（每个 CTA 逐个查）**：白底白字、透明按钮贴同色背景无描边、
  幽灵按钮压在花哨背景上无 scrim → 全部违规；大字 3:1、正文字号 4.5:1 起
- **CTA 不换行**：主按钮文案 ≤ 3 词/6 个汉字，桌面端换行即失败
- **同一意图只留一个 CTA 文案**：「联系我 / 开始合作 / 聊一聊」是同一意图，
  全页统一成一个说法
- **正文规格**：中文行高 ≥ 1.7（暗底建议 1.8）、每行 18-40 字、`max-width` 锁行长；
  展示巨字与正文段之间留 ≥ 1.5 倍常规间距，不许巨字挤压正文
- **形状一致性锁**：全页一套圆角刻度（全直角 / 全 12-16px / 全 pill），
  混用必须有成文规则并全页执行
- **文案自审**：交付前重读页面上每一句可见文字（含按钮/眉批/脚注），
  语法不通、指代不明、"AI 装深沉"的句子一律改成朴素直白的话；
  编不圆的"精确假数据"（4.1×、5.8mm）要么来自简报要么删

## 三、内容密度（首页是第一印象，不是全文阅读）

- 每节默认形状：短标题（≤ 8 词）+ 短段（≤ 25 词）+ 一个视觉资产或一个 CTA
- 列表 > 5 项就换组件：双栏分组 / 卡片网格 / 标签页 / 横向 scroll-snap /
  marquee——默认 `<ul>` + divide-y 长表是懒惰
- 规格/参数表禁止每行一条 hairline 到底：改 2 列卡片（大数字 + 一句为什么重要）、
  分 3 组稀疏分隔、或"3-4 个主打大瓦片 + 其余折叠"

## 四、多方向展示规范（多样性任务专用）

同页展示 N 个设计方向时（如 3 个播放器方向）：

- **每个方向一条全宽沉浸带**：自带背景环境、氛围层与配套排版，进入该带即进入
  该方向的世界——禁止三张小卡并排挤在同一中性底上
- **每个方向独立过工艺密度 ≥ 3**（含至少一个真实可交互细节），
  小尺寸不是低工艺的借口
- 方向之间轴级差异照旧（divergence-playbook），且**每方向配一句设计宣言**
  （它像什么杂志/空间/年代）直接排进版面，作为方向的一部分

## 五、评审视角扩充（两轮制第 4 视角）

craft-loop 三视角之外增加**读者视角**：
眯眼通读页面全部正文 30 秒——哪一段最费劲？哪个按钮看不清？哪行字被纹理/
巨字/低对比度干扰？至少挑出一处，与其他视角一并进第二轮修复。
