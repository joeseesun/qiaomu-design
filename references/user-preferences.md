# 用户偏好账本（User Preferences Ledger）

> 本文件由「自进化机制」维护：用户在使用中给出的反馈被抽象成规则后追加于此。
> **每次执行任何设计任务前必读；preflight 门禁第 0 步逐条对照本文件。**
> 与通用规则冲突时，本文件优先。

## 规则格式

```
### P-{编号} · {规则一句话}
- 级别：硬禁令 / 强偏好 / 情境规则
- 何时适用：{范围}
- 原话摘录："{用户当时的表述}"
- 收录：{日期}｜状态：生效 / 已废止（{日期，原因}）
```

---

### P-1 · 界面排版禁用斜体，中文尤其
- 级别：硬禁令
- 何时适用：一切 UI/页面产出；强调改用字重、颜色、字号
- 原话摘录："界面排版中，出现斜体会很不好看，不和谐。"
- 收录：2026-07-03｜状态：生效（已落入 SKILL.md 字体禁令 + preflight A 节）

### P-2 · 设计方向必须"先看后选"
- 级别：强偏好（已升级为招牌机制）
- 何时适用：任何涉及视觉方向的任务，生成风格试衣间预览页（4 方向 + 推荐 + 60s 自动推进）
- 原话摘录："每次设计时，都快速给用户几种风格预览（用本地 web 页面弹出），用户选中方向后开始设计。"
- 收录：2026-07-03｜状态：生效（已落入 Phase 2 + style-preview.md）

### P-4 · Hero 视觉侧不得简陋
- 级别：硬禁令
- 何时适用：任何分屏 Hero / 产品展示位；空心 div mockup、文字+渐变斑点均违规，按 assets-and-readability.md 三选一执行
- 原话摘录："hero 区右侧设计很简陋"
- 收录：2026-07-04｜状态：生效

### P-5 · 可读性优先于气质
- 级别：硬禁令
- 何时适用：一切含正文的页面；暗底正文 .85/次要 ≥.60、正文区纹理 ≤3%、CTA 对比度逐个审计
- 原话摘录："个人作品集可读性（差）"
- 收录：2026-07-04｜状态：生效

### P-6 · 多方向展示每个方向都要是完整世界
- 级别：强偏好
- 何时适用：同页展示多个设计方向时：全宽沉浸带 + 每方向独立工艺密度 ≥3 + 设计宣言入版面，禁小卡并排糊弄
- 原话摘录："多样性测试效果很差"
- 收录：2026-07-04｜状态：生效

### P-7 · 创意不许吃掉页面的功能契约
- 级别：硬禁令
- 何时适用：一切页面，开放创意命题尤其；页面类型的核心标识必须首屏可辨（404 的错误码、定价的价格、表单的提交路径）
- 原话摘录："我们 skill 做的 404 页面，为啥连 404 都没出现，虽然创意很好，但会不会有点稍微走远了一点。"
- 收录：2026-07-04｜状态：生效（已落入 SKILL.md 功能契约 + preflight A4 + 甲方评审视角）

### P-3 · 装饰性中文 webfont 严禁滥用
- 级别：硬禁令
- 何时适用：一切含中文的页面；正文/UI 系统栈，装饰字体仅创意标题且 text= 子集化
- 原话摘录："用户大概率没有装这些字体，如果用 cdn 会有速度影响。只有个别场景可以用，大多数场景千万别滥用。"
- 收录：2026-07-03｜状态：生效（已落入 chinese-typography.md 第一硬规则 + preflight D2）
