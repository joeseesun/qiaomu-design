# 工艺环（Craft Loop）：抬上限的三个机制

> 来源：v2.4 PK 复盘——禁令和门禁只能抬下限（不翻车），不能产生上限（惊艳）。
> 本文件是 Phase 3 的进攻手册：DNA 注入 → 工艺密度 → 两轮制。三者全部强制。

## 一、参考 DNA 注入（先偷再做）

方向确定后、写码前，从 `references/design-systems/{58 站}/DESIGN.md` 中挑
**1-2 个气质最接近的站点**作为 DNA 供体，提取 **3-5 个具体的值或动作**写进设计计划：

- 不是"参考 Stripe 的感觉"，而是"偷 Stripe 的三层染色阴影栈
  `rgba(...) 0 0 0 1px, ... 0 2px 6px, ... 0 4px 8px`"
- 可偷的东西：阴影体系、圆角刻度、字阶比例、hover 位移量、边框策略、
  导航结构、留白节奏、色彩角色分配
- 设计计划里必须逐条写明"从 {site} 偷了什么"——写不出来就是没偷，重来

**为什么强制**：58 站库是本 skill 独有资产，闲置等于没有。大师的具体数值
比自己拍脑袋的数值好，且可溯源。

## 二、工艺密度（Craft Density ≥ 5）

平庸和惊艳的差距不在布局，在**每屏有意为之的细节数量**。
每个页面必须从下面菜单落地 **至少 5 项**（交付时逐项报告）：

| # | 签名动作 | 示例 |
|---|---------|------|
| 1 | 氛围层 | body::before 噪点/网格线/渐变雾/纸纹（透明度 <6%） |
| 2 | ::selection 定制 | 选中文本用品牌色底 |
| 3 | 品牌 focus 环 | focus-visible 用品牌色 + 偏移，不用浏览器默认蓝 |
| 4 | 签名交互 | 磁吸按钮 / hover 揭示 / 滚动显现 / 数字滚动，**一个足矣** |
| 5 | 多层染色阴影 | 2-3 层、染底色色调，拒绝单层灰阴影 |
| 6 | 编辑细节 | 章节编号（01/02）、页边注、hairline、大写字距标签 |
| 7 | 展示排版动作 | 超大字/描边字/垂直排字/文字与图形重叠 |
| 8 | 流体字号 | 标题 clamp(min, vw, max) |
| 9 | 自定义滚动条 | 细窄 + 品牌色 thumb（配合主题） |
| 10 | 收尾彩蛋 | 页脚一行有人味的小字 / hover 才出现的细节 |
| 11 | 图形资产 | 手绘感 SVG 插图/图标，拒绝 emoji 和通用图标堆 |
| 12 | 材质细节 | 内边框高光、glass 折射边、纸张齿孔、印刷套准偏移 |

约束：动作要服务方向气质（终端风就别上纸纹），单页签名交互 ≤1，
氛围层透明度低于 6%——密度不等于吵闹。

## 三、两轮制（Render → Critique → Refine）

**一轮交付是赌运气，两轮交付才是设计。** 流程：

1. **第一轮**：按方向 + DNA + 工艺密度做出完整页面
2. **渲染**：用浏览器工具真实打开页面，截桌面（≥1280px）截图；
   若环境无浏览器，逐段通读代码模拟渲染
3. **四视角偏执评审**（各挑一处最弱，共 4-6 处，写下来）：
   - **艺术总监**：哪一屏最平庸、最像模板？记忆点成立吗？Hero 视觉侧配得上
     标题吗（空心 mockup / 渐变斑点 = 必修）？
   - **工程师**：哪里对不齐/溢出/状态缺失/间距不一致？
   - **甲方**：三秒看完，我记住了什么？（答不上来 = 记忆点失败，必修）
     以及：**这页的功能契约完成了吗**——用户来这页要完成的事，3 秒内能开始吗？
     （404 认不出是报错页、定价没法比价 = 契约违约，必修，优先级高于一切美学问题）
   - **读者**：眯眼通读全部正文 30 秒——哪段最费劲？哪个按钮看不清？哪行字被
     纹理/低对比度干扰？（详见 assets-and-readability.md 可读性红线）
4. **第二轮**：**只修这 3-5 处**，禁止推倒重做、禁止顺手大改
5. 重新截图确认修复生效 → 过 preflight → 交付

评审要苛刻：三个视角**必须各挑出至少一处**，"没毛病"不是合法答案——
偏执人格的存在意义就在这一步。

## 四、优先级声明

时间或篇幅受限时的取舍顺序：**两轮制 > DNA 注入 > 工艺密度凑满 5 项**。
门禁（preflight）永远不可省略，但记住：门禁保证及格，本文件负责高分。
